# Метод опорных векторов SVM и оценка качества моделей

Репозиторий содержит цикл практических работ и лабораторных заданий, посвященных изучению и применению Метода опорных векторов (Support Vector Machine — SVM), подбору гиперпараметров с помощью кросс-валидации, а также анализу качества классификации с использованием метрик F1-score и ROC-AUC.

#### 1. 7_1_SVM(Метод Опорных векторов).ipynb
* **Суть работы:** Знакомство с базовым алгоритмом SVM на примере классического датасета load_wine (классификация вин) из библиотеки scikit-learn.
* **Что делается:**
  * Проводится разведочный анализ данных (проверка баланса классов).
  * Выполняется стандартизация признаков при помощи StandardScaler.
  * Обучается линейная модель SVC.
  * Оценивается качество классификации на тестовой выборке с упором на метрику F1-score (так как исходные классы несбалансированы).

#### 2. 7_2_SVM с ядрами.ipynb
* **Суть работы:** Изучение нелинейного SVM и концепции Kernel Trick (ядерного трюка).
* **Что делается:**
  * Генерируются сложные нелинейные синтетические данные (make_circles, make_moons).
  * Демонстрируется ограничение линейного ядра (Linear Kernel), которое не может разделить такие данные.
  * Применяются нелинейные ядра (например, RBF — радиально-базисная функция).
  * Визуализируются разделяющие поверхности (Decision Regions) с помощью библиотеки mlxtend для наглядного понимания работы ядер.

#### 3. 7_3_ROC_AUC.ipynb
* **Суть работы:** Глубокое погружение в оценку качества моделей бинарной классификации через вероятностные метрики.
* **Что делается:**
  * Задача классификации вина сводится к бинарной (класс 1 против всех остальных).
  * Модель SVM настраивается на выдачу вероятностей (параметр probability=True).
  * Строится график ROC-кривой (зависимость True Positive Rate от False Positive Rate).
  * Рассчитывается метрика ROC-AUC (Area Under Curve) для оценки предсказательной способности алгоритма.

#### 4. 7_4_Дз SVM+CrossVal.ipynb
* **Суть работы:** Итоговое практическое (домашнее) задание на реальном датасете adult.csv (прогнозирование уровня дохода человека более 50k долларов).
* **Что делается:**
  * **Препроцессинг:** Поиск и обработка пропущенных значений (обозначенных как знак вопроса).
  * **Оптимизация:** Для ускорения обучения ресурсоемкого SVM применяется библиотека scikit-learn-intelex (sklearnex).
  * **Эксперименты:** Реализуется подбор гиперпараметров (таких как параметр регуляризации C и коэффициенты ядер) с использованием K-Fold кросс-валидации.
  * Модели оцениваются и сравниваются между собой по метрике ROC-AUC.
